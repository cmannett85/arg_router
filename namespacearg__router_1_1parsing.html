Visitation pattern to find a named child.

Visitor needs to be equivalent to:

template <typename Child
operator()(std::size_t i, Child&&, match_result) {}

i is the index into the children tuple. If child has a match(std::string_view) method then the visitor is called if a match is found. If it doesn't have a match method and the prefix is prefix_type::NONE then every child encountered that fits that description will have visitor called on it - in that circumstance match_result::matched will be match_result::NO_MATCH.
