Positional arg-aware visitation pattern to find a named child.

This is the same as other visit_child implementation except that it uses the already parsed values and hit mask to determine which positional arg-like child visitor should operate on. The visitor is not called more than once per invocation.

Positional args may accept multiple values, in which case the child that represents it will be used to parse the value, until that positional arg's maximum_count() or count() is reached - at which point the next positional arg is used.

template<typename ChildrenTuple , typename Fn >
bool arg_router::parsing::visit_child ( const token_type &  token,
const ChildrenTuple &  children,
Fn  visitor 
)

Visitation pattern to find a named child.

Visitor needs to be equivalent to:

template <typename Child>
operator()(std::size_t i, Child&&) {}

i is the index into the children tuple. If child has a match(std::string_view) method then the visitor is called if a match is found. If it doesn't have a match method and the prefix is prefix_type::NONE then every child encountered that fits that description will have visitor called on it.
